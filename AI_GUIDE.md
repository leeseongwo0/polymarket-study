# AI 가이드 — 학습자가 이 레포를 클론한 뒤 AI가 참고할 규칙

이 문서는 학습자가 AI에게 “이 데모를 만들어줘/고쳐줘/설명해줘”라고 지시할 때 AI가 따라야 하는 기준입니다. 자동 코딩 에이전트는 `AGENTS.md`를 우선 읽고, 이 문서를 함께 참고하세요.

## 한 줄 목표

초보자가 하루 안에 이해할 수 있는 **Polymarket 영감의 안전한 시뮬레이션 데모**를 유지·확장합니다.

## AI가 반드시 지킬 것

1. **범위 유지**
   - 플레이 크레딧, 목업 지갑, 로컬 상태만 사용합니다.
   - 실제 돈, 메인넷, 지갑 서명, 예치/출금, 실거래, 투자 조언은 추가하지 않습니다.

2. **작은 단위 작업**
   - 한 번에 하나의 기능 또는 하나의 버그만 다룹니다.
   - 수정 전 “바꿀 파일 후보”를 짧게 말합니다.
   - 수정 후 “무엇이 왜 바뀌었는지”를 초보자 언어로 설명합니다.

3. **검증 우선**
   - 순수 로직 변경: `npm run test`
   - UI/라우팅 변경: `npm run lint`, `npm run build`, 필요 시 `npm run test:e2e`
   - 문서 변경: 링크와 명령이 현재 파일 구조와 맞는지 확인합니다.

4. **학습 설명 포함**
   - 코드를 생성한 뒤 파일별 역할을 설명합니다.
   - 테스트가 보호하는 동작을 설명합니다.
   - 실패 로그가 있으면 첫 번째 오류부터 해석합니다.

5. **테스트 삭제 금지**
   - 테스트가 실패하면 동작, 셀렉터, 기대값 중 무엇이 틀렸는지 진단합니다.
   - 학습-critical 테스트를 삭제하거나 약하게 만들어 통과시키지 않습니다.

## AI가 하지 말아야 할 것

- `yarn`, `pnpm`, 새 프레임워크, 새 DB, 새 인증 시스템을 임의로 추가하지 않습니다.
- 실제 Polymarket API나 실거래 인프라를 붙이지 않습니다.
- USD, 수익, 베팅, 투자 추천처럼 오해될 수 있는 문구를 UI 핵심 흐름에 넣지 않습니다.
- Polymarket 로고/이미지/브랜드 자산을 복사하지 않습니다.
- “프로덕션 수준 예측시장”이라고 주장하지 않습니다.

## 추천 응답 형식

```text
목표: <이번 작업으로 사용자가 얻을 결과>
바꿀 파일: <예상 파일 1~4개>
작업: <구현/수정 요약>
검증: <실행한 명령과 결과 또는 다음에 실행할 명령>
학습 포인트: <초보자가 이해해야 할 핵심>
```

## AI에게 넣어도 좋은 시스템/상단 프롬프트

```text
이 레포는 하루짜리 Polymarket 메커니즘 스터디입니다. AGENTS.md, AI_GUIDE.md, LEARNER_GUIDE.md, docs/demo-1-prd-guide.md를 기준으로 작업하세요. 실제 돈, 메인넷, 실거래, 투자 조언, Polymarket 공식 제휴/브랜드 복사는 금지입니다. npm만 사용하고, 작은 diff로 수정한 뒤 검증 명령을 알려주세요.
```
