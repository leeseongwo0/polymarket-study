# 안전과 범위

이 프로젝트는 하루짜리 교육용 예측시장 메커니즘 실험실입니다. 금융 상품도, 프로덕션 예측시장 서비스도 아닙니다.

## 강한 비목표

- 메인넷 없음
- 실제 돈, 실제 자산, 실제 베팅, 실거래 없음
- 실제 세계 자산 정산 없음
- 금융, 투자, 법률, 세무 조언 없음
- Polymarket 공식 제휴 없음
- Polymarket 로고, 상표, 브랜드 자산, 픽셀 단위 복제 없음
- v1에서 프로덕션 CLOB, 매칭 엔진, 커스텀 스마트컨트랙트, OAuth, RBAC 없음

## 대신 앱이 하는 것

- 플레이 크레딧을 사용합니다.
- 시드 학습용 마켓을 사용합니다.
- 장난감 확률 모델을 사용합니다.
- 모의 정산을 사용합니다.
- 네트워크에 연결하지 않는 목업 지갑 실험을 사용합니다.

## 이 경계가 중요한 이유

초보자는 프로덕션에 가까운 금융·web3 시스템을 다루기 전에 원리를 안전하게 배워야 합니다. 이 제약은 하루짜리 학습 루트를 현실적으로 만들고 법적, 안전, 설정 리스크를 줄입니다.

## 확장 아이디어를 다룰 때의 원칙

- 실제 지갑 대신 mock wallet으로 먼저 UX를 설명합니다.
- 실제 돈 대신 play credits로 상태 변화를 설명합니다.
- 실제 API 대신 seed data와 read-only route handler를 사용합니다.
- production realism보다 학습 가능성과 검증 가능성을 우선합니다.
