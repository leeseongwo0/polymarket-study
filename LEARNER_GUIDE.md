# 학습자 가이드 — AI에게 어떻게 명령할까?

이 레포는 “AI가 다 만들어 주는지 보기”가 아니라, **AI를 페어 프로그래머처럼 쓰면서 내가 제품 요구사항, 코드 변화, 검증 증거를 이해하는 연습**을 하기 위한 스터디입니다.

## 학습자가 지킬 5가지 규칙

1. **한 번에 하나만 요청하기**
   - 나쁜 예: “폴리마켓 클론 전부 만들어줘.”
   - 좋은 예: “YES를 누르면 확률과 포트폴리오가 바뀌는 순수 함수와 테스트를 만들어줘.”

2. **PRD를 먼저 주기**
   - [`docs/demo-1-prd-guide.md`](docs/demo-1-prd-guide.md)의 “AI에게 전달할 PRD 블록”을 붙여 넣고 시작하세요.

3. **AI에게 파일 후보와 검증 명령을 요구하기**
   - “수정 전에 바꿀 파일 후보를 알려줘.”
   - “수정 후 어떤 npm 명령으로 확인해야 하는지 알려줘.”

4. **diff를 읽고 실행하기**
   - AI가 만든 코드를 그대로 믿지 말고 변경된 파일을 읽습니다.
   - 최소한 `npm run test` 또는 AI가 제안한 검증 명령을 실행합니다.

5. **실패 로그를 정확히 붙여 넣기**
   - “안 돼요” 대신 명령, 전체 오류, 직전에 바꾼 파일을 알려줍니다.

## 첫 프롬프트 예시

```text
이 레포는 하루짜리 Polymarket 메커니즘 스터디입니다. AGENTS.md, AI_GUIDE.md, LEARNER_GUIDE.md, docs/demo-1-prd-guide.md를 먼저 읽고 작업해줘.

목표: 완성 데모 1의 구조를 이해하고 싶어.
요청: src/app, src/components, src/lib, tests, e2e가 각각 무슨 역할인지 초보자 눈높이로 설명해줘. 아직 코드는 바꾸지 마.
```

## 기능 구현 프롬프트 예시

```text
목표: 마켓 카드에 learningGoal을 짧게 보여주고 싶어.
제약: 실제 돈/투자 조언/브랜드 복사는 금지, 새 의존성 금지, npm만 사용.
요청: 수정 전에 바꿀 파일 후보를 알려주고, 작은 diff로 구현해줘. 수정 후 실행할 검증 명령도 알려줘.
```

## 테스트 요청 프롬프트 예시

```text
src/lib/market-engine.ts를 바꾸기 전에 현재 동작을 보호하는 Vitest 테스트를 먼저 추가해줘. YES 구매, NO 구매, 잔액 초과, 정산 후 거래 금지 중 빠진 케이스가 있는지 확인해줘.
```

## 디버깅 프롬프트 템플릿

```text
명령: npm run build
관찰한 출력:
<첫 번째 전체 오류 블록 붙여 넣기>
기대값: 프로덕션 빌드 성공
바로 직전에 바꾼 것: <파일 또는 기능>
요청: 가장 가능성 높은 원인, 가장 작은 수정, 수정 뒤 다시 실행할 명령을 알려줘.
```

## 내가 직접 이해해야 하는 파일

| 파일 | 왜 중요한가 |
| --- | --- |
| `src/lib/market-engine.ts` | YES/NO 선택이 확률, 잔액, 포지션을 어떻게 바꾸는지 담은 핵심 로직입니다. |
| `tests/unit/market-engine.test.ts` | 핵심 로직이 망가지지 않았다는 증거입니다. |
| `src/components/market/TradePanel.tsx` | 사용자가 실제로 누르는 거래/정산 UI입니다. |
| `src/lib/portfolio-store.ts` | 새로고침 후에도 데모 상태가 남는 이유를 보여줍니다. |
| `e2e/market-flow.spec.ts` | 브라우저에서 학습자 흐름이 실제로 되는지 확인합니다. |
| `AGENTS.md` | AI가 이 레포에서 지켜야 하는 안전·기술 규칙입니다. |

## 중간 체크리스트

- [ ] `npm install`이 끝났다.
- [ ] `npm run dev`로 데모를 열었다.
- [ ] 홈에서 마켓을 클릭해 상세 페이지로 이동했다.
- [ ] Buy Yes 또는 Buy No 후 확률/잔액/히스토리가 바뀌는 것을 봤다.
- [ ] Resolve Yes/No 후 모의 정산 메시지를 봤다.
- [ ] 목업 지갑이 실제 지갑이 아니라는 문구를 확인했다.
- [ ] `npm run test`를 실행했다.
- [ ] 오늘 바꾼 내용을 내 말로 3문장 이상 설명할 수 있다.

## 멈춰야 하는 신호

AI가 아래를 제안하면 그대로 진행하지 말고 범위를 다시 말하세요.

- “실제 지갑을 연결하자”, “메인넷 RPC를 넣자”
- “USD/수익/베팅/투자 수익률을 표시하자”
- “테스트가 귀찮으니 삭제하자”
- “DB/Auth/결제를 붙이면 더 완성도 있다”
- “Polymarket 로고를 가져오자”

대신 이렇게 말합니다.

```text
이 스터디는 하루짜리 교육용 시뮬레이션이야. 실제 돈, 메인넷, 브랜드 자산, 실거래는 범위 밖이야. 목업 또는 play-credit 방식으로 대체해줘.
```
